Explanation of the different concepts:

  • Category(name): represents a type of food (meat, vegetable,...).
  • Food(name, id_category, information): represents a food, is linked to a category and contains some information.
  • Menu(name, information, actif): represents a Menu, contains information and can be active or no more usable.
  • Composed(id_menu, id_food, quantity): represents a component of a Menu.
  • Version(name, release_date): represents a version of a Tray.
  • Tray(name, id_version, information, ip, online, actif, on_use, timestamp): represents the system that collected the data, it has a version, an ip address. It may be on_used or free.
  • Meal(id_tray, id_user, id_menu, start, end, actif, information): represent an experience a user made on a specific tray. That "start" and "end" at a specific time. If an error occurs, you may put active to false.
  • User(name, email, password, weight, actif, birthdate, gender, id_permission): represents a user, with some information about it. An email address and a password is use to sign in the app. And some permission in the website.
  • Permission(name): represents some permission in the website.
